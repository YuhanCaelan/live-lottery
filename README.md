# live-lottery

一个用于晚会、大屏幕和现场活动的纯前端抽奖页面。项目只包含 HTML、CSS 和 JavaScript，不需要安装依赖，也不需要启动服务，适合交给非技术人员长期使用。

## 功能概览

- 大屏抽奖展示，适合现场投屏使用
- 支持设置抽奖轮数和每轮中奖人数
- 支持一层、二层座位参与控制
- 支持单个座位点击排除或恢复
- 支持按楼层、排、列批量排除座位范围
- 支持自定义抽奖页标语
- 支持自定义背景图片和 BGM
- 支持抽奖结果展示、重置和本地记忆配置

## 文件说明

- `index.html`：页面结构，双击即可打开
- `style.css`：界面样式和动效
- `script.js`：抽奖逻辑、座位控制和设置保存
- `assets/bg.jpg`：默认背景图，可按需替换或补充

## 快速使用

1. 下载或复制整个项目文件夹。
2. 双击打开 `index.html`。
3. 点击右上角“设置”。
4. 设置抽奖轮数、每轮人数、抽奖页标语和座位排除范围。
5. 点击“开始抽奖”开始滚动。
6. 点击“停止抽奖”锁定本轮中奖结果。
7. 抽完后可点击“显示全部抽奖结果”查看汇总。
8. 如需重新开始，点击“重置抽奖”。

## 设置说明

### 抽奖轮数

设置一共抽几轮。例如设置为 `2`，表示抽两轮。

### 每轮人数

设置每轮抽出几个人。例如设置为 `3`，表示每轮抽出 3 个座位。

### 抽奖页标语

用于修改主舞台中默认显示的“今天，你就是C位”。输入后会立即生效，并保存到当前浏览器。

### 启用二层座位

勾选后，二层座位参与抽奖；取消勾选后，二层座位不参与抽奖。

### 单个座位排除

打开设置页后，点击座位图中的座位即可切换：

- 蓝色：参与抽奖
- 橙色：不参与抽奖
- 黄色：已中奖
- 灰色：当前楼层禁用

### 快速排除

在“快速排除”区域填写楼层、起始排、结束排、起始列、结束列，然后点击“排除范围”。适合一次排除一片区域。

## 媒体设置

### 背景图片

点击“选择背景图片”，选择本机图片文件。背景图片会保存到当前浏览器的本地存储中，刷新页面后仍然有效。

建议：

- 使用 JPG、PNG、WebP 等常见图片格式
- 图片大小尽量控制在 10MB 以内
- 如果图片过大，浏览器可能无法保存，会自动回退到默认背景

### BGM

点击“选择 BGM”，选择本机音频文件。BGM 会在当前打开的页面中使用。

注意：

- BGM 只在当前打开页面内有效
- 刷新页面或重新打开页面后，需要重新选择 BGM
- 这是浏览器安全限制导致的正常现象

### 恢复默认媒体

点击“恢复默认媒体”后，会清除自定义背景和当前 BGM，恢复默认舞台状态。

## 数据保存规则

本项目是纯静态前端应用，所有设置保存在当前浏览器中：

- 抽奖轮数
- 每轮人数
- 抽奖页标语
- 座位排除状态
- 是否启用二层座位
- 当前浏览器保存的背景图片

以下内容不会跨浏览器或跨设备同步：

- 用户选择的 BGM
- 其他电脑或其他浏览器中的设置

如果清理浏览器数据，设置会恢复默认。

## 部署方式

### 本地使用

直接双击 `index.html` 即可。

## 常见问题

### 为什么换电脑后自定义背景不见了？

因为本项目是纯前端静态页面，自定义背景保存在当前浏览器本地。换电脑、换浏览器或清理浏览器数据后，需要重新选择。

### 为什么 BGM 刷新后不见了？

BGM 使用浏览器临时文件地址播放，刷新页面后临时地址会失效。这是为了保持项目纯静态、无需服务器。

### 可以长期跨设备保存自定义媒体吗？

纯 HTML、CSS、JavaScript 无法做到。跨设备长期保存媒体需要服务器上传接口或对象存储服务。

### 为什么点击设置页座位图时没有明显卡顿？

座位图已做性能优化：座位状态增量更新、批量更新分帧处理、设置页滚动时暂停舞台动画。

## 浏览器建议

- 推荐使用 Chrome、Edge 或其他现代桌面浏览器
- 建议在正式活动前提前打开页面并完成设置
- 建议活动前测试一次背景、BGM、座位排除和抽奖流程
